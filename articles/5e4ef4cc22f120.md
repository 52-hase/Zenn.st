# アーキテクチャ整理

① フロントエンド
- JavaScript
・イベント検知
・DOM更新
・WebSocket通信

- Bootstrap
・UI整形
・レスポンシブ対応

- Rails View
・画面描画（HTML生成）

② バックエンド（Heroku上）
- Ruby 3.2.2 / Rails 7.1.3
・MVC構成

役割
・Controller：リクエスト受付
・Model（ActiveRecord）：DB操作・バリデーション
・View：レスポンス生成

③ リアルタイム通信
- ActionCable（WebSocket）
・双方向通信
・ルーム単位ブロードキャスト
・ページ更新なしで同期

④ データベース
- PostgreSQL（Heroku Postgres）
・ユーザー
・メッセージ
・再生状態

⑤ ファイル保存
- Active Storage
- AWS S3
・画像・アップロードファイル保存

⑥ 外部API
- Spotify Web API（rspotify）
・楽曲情報取得
・再生制御

⑦ 開発環境
- Docker
・Rails / PostgreSQL / Node.js をコンテナ化
・環境差異を排除

- メッセージ送信フロー（最重要）
```
① ユーザー入力
  ↓
② JavaScript（送信）
  ↓
③ ActionCable（WebSocket）
  ↓
④ Rails Channel
  ↓
⑤ ActiveRecord
  ↓
⑥ PostgreSQL保存
  ↓
⑦ ActionCableブロードキャスト
  ↓
⑧ JavaScript（DOM更新）
  ↓
全員に即時反映
```
- ポイント
・HTTPではなくWebSocket
・DB保存 → 全員へ配信
・ページリロード不要

- Spotify再生同期フロー
```
① 再生ボタン押下
  ↓
② JavaScript
  ↓
③ ActionCable送信
  ↓
④ Rails Channel
  ↓
⑤ Spotify Web API
  ↓
⑥ 再生状態をDB保存
  ↓
⑦ ルームへブロードキャスト
  ↓
⑧ 各ブラウザでUI同期
```

再生状態モデル（必要カラムのみ）
```
spotify_track_id: Spotifyとの紐付け
title: 表示用
artist_name: 表示用
started_at: 同期基準時刻
room_id: ルーム管理
```

# 総合的に作成した

画像の説明
### ① フロントエンド
ブラウザ上で動作し、ユーザーの操作を受け取る部分

JavaScript
役割
ユーザー操作を検知し、画面を動的に更新する
具体的な処理
メッセージ送信ボタンなどのイベント検知
新しいメッセージや再生情報をDOM更新
WebSocket通信でサーバーとリアルタイム通信

解説
JavaScriptによって、ページをリロードせずに
チャット表示やSpotifyの再生状態をリアルタイムで更新できます。

Bootstrap
役割
UIデザインを効率よく整える
具体的な処理
ボタン・カードなどのUIコンポーネント提供
レスポンシブ対応

解説
Bootstrapを利用することで
PC・スマホ両方で見やすいUIを少ないコード量で実装できます。

Rails View
役割
HTMLを生成して画面を表示
具体的な処理
ERBテンプレートによるHTML生成
サーバーから受け取ったデータを画面に表示

解説
Rails Viewはサーバー側でHTMLを生成し、
ブラウザへレスポンスとして返します。

### ② バックエンド（Heroku）
サーバー側で動作するアプリケーション
Ruby 3.2.2 / Rails 7.1.3
RailsのMVC構成を利用してアプリケーションを構築しています。

Controller
役割
ユーザーのリクエストを受け取る
処理をModelへ指示
具体的な処理
HTTPリクエスト受付
Spotify API呼び出し
DB操作の指示
Viewへのデータ受け渡し
Model（ActiveRecord）

役割
データ管理
具体的な処理
DB操作（CRUD）
バリデーション
アソシエーション管理

例
保存しているデータ
User
Message
PlayingTrack（再生状態）
View

役割
ユーザーへ返すレスポンス生成
具体的な処理
HTML生成
データ表示

### ③ リアルタイム通信
ActionCable（WebSocket）
役割
ブラウザとRailsサーバーの
双方向リアルタイム通信
できること
チャットの即時更新
Spotify再生状態の同期
ルーム単位の配信

解説
通常のHTTP通信は
「リクエスト → レスポンス」の一方向ですが
WebSocketでは
クライアント ⇄ サーバー
の双方向通信が可能になります。
そのため
チャット
再生同期
などのリアルタイム機能を実装できます。

### ④ データベース
PostgreSQL（Heroku Postgres）
役割
アプリのデータを永続化する。
保存しているデータ
ユーザー情報
チャットメッセージ
再生状態

解説
RailsのActiveRecordを通して
DBにデータを保存・取得しています。

### ⑤ ファイル保存
Active Storage + AWS S3
役割
画像やファイルの保存
構成
Rails
  ↓
ActiveStorage
  ↓
AWS S3

解説
アプリサーバーではなく
クラウドストレージ（S3）に保存することで
スケーラビリティ向上
サーバー負荷軽減
が可能になります。

### ⑥ 外部API
Spotify Web API
用途
楽曲情報取得
楽曲再生
再生状態取得
実装
Railsから
rspotify
というgemを利用して
Spotify APIを呼び出しています。

### ⑦ 開発環境
Docker
役割
開発環境をコンテナ化して
環境差異をなくす。
コンテナ化しているもの
Rails
PostgreSQL
Node.js

メリット
環境構築が簡単
チーム開発で環境差異が発生しない
再現性の高い開発環境

### ⑧ メッセージ送信の流れ
ユーザーがメッセージ入力
        ↓
JavaScriptが送信処理
        ↓
WebSocketでActionCableへ送信
        ↓
Rails Channelが受信
        ↓
MessageをDB保存
        ↓
ActionCableがルームへブロードキャスト
        ↓
各ユーザーのJavaScriptが受信
        ↓
DOM更新して即時表示

解説
WebSocketを利用することで
ページ更新なしでチャットを全ユーザーへ即時表示できます。

⑨ Spotify再生同期の流れ
再生ボタン押下
        ↓
JavaScriptが再生リクエスト送信
        ↓
WebSocketでRailsへ送信
        ↓
Rails Channelが処理
        ↓
Spotify Web API呼び出し
        ↓
再生状態をDB保存
        ↓
ActionCableがルームへ配信
        ↓
全ユーザーの再生UIが同期
