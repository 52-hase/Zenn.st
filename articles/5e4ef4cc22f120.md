# アーキテクチャ整理

① フロントエンド
- JavaScript
・イベント検知
・DOM更新
・WebSocket通信

- Bootstrap
・UI整形
・レスポンシブ対応

- Rails View
・画面描画（HTML生成）

② バックエンド（Heroku上）
- Ruby 3.2.2 / Rails 7.1.3
・MVC構成

役割
・Controller：リクエスト受付
・Model（ActiveRecord）：DB操作・バリデーション
・View：レスポンス生成

③ リアルタイム通信
- ActionCable（WebSocket）
・双方向通信
・ルーム単位ブロードキャスト
・ページ更新なしで同期

④ データベース
- PostgreSQL（Heroku Postgres）
・ユーザー
・メッセージ
・再生状態

⑤ ファイル保存
- Active Storage
- AWS S3
・画像・アップロードファイル保存

⑥ 外部API
- Spotify Web API（rspotify）
・楽曲情報取得
・再生制御

⑦ 開発環境
- Docker
・Rails / PostgreSQL / Node.js をコンテナ化
・環境差異を排除

- メッセージ送信フロー（最重要）
```
① ユーザー入力
  ↓
② JavaScript（送信）
  ↓
③ ActionCable（WebSocket）
  ↓
④ Rails Channel
  ↓
⑤ ActiveRecord
  ↓
⑥ PostgreSQL保存
  ↓
⑦ ActionCableブロードキャスト
  ↓
⑧ JavaScript（DOM更新）
  ↓
全員に即時反映
```
- ポイント
・HTTPではなくWebSocket
・DB保存 → 全員へ配信
・ページリロード不要

- Spotify再生同期フロー
```
① 再生ボタン押下
  ↓
② JavaScript
  ↓
③ ActionCable送信
  ↓
④ Rails Channel
  ↓
⑤ Spotify Web API
  ↓
⑥ 再生状態をDB保存
  ↓
⑦ ルームへブロードキャスト
  ↓
⑧ 各ブラウザでUI同期
```

再生状態モデル（必要カラムのみ）
```
spotify_track_id: Spotifyとの紐付け
title: 表示用
artist_name: 表示用
started_at: 同期基準時刻
room_id: ルーム管理
```
