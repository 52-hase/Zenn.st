## TypeScriptとは？

#### TypeScriptは、JavaScriptに「型」の仕組みを追加したプログラミング言語
・Microsoftが開発したオープンソース
・JavaScriptのスーパーセット（上位互換）
・JavaScriptのコードはそのまま使える

型を導入することで、エラーの防止・安全性・可読性の向上が可能になる
そのため、
・フロントエンド（React / Vue など）
・バックエンド（Node.js）
といった幅広い開発で利用されている

### TypeScriptの役割と重要性
・JavaScriptは動的型付け言語のため、
・型の間違いに実行時まで気づけないという弱点がある

TypeScriptを使うことで、
・コンパイル時に型チェックが行われる
・実行前にエラーを発見できる
・大規模・チーム開発でも管理しやすい
といったメリットがある

### JavaScriptとの関係性・違い
JavaScriptとの関係
・TypeScriptは最終的に**JavaScriptに変換（コンパイル）**される
・JavaScriptの機能・ライブラリはすべて利用可能
・既存のJavaScriptから段階的に移行できる

大きな違い
・静的型付けがあるかどうか

言語と型の扱い
・JavaScript	動的型付け
・TypeScript	静的型付け
TypeScriptでは、実行前にエラーを防げる点が最大の違い

### TypeScriptの基本概念
コンパイルとは？
TypeScriptはそのまま実行できない
・ts ファイル → コンパイル
・js ファイルに変換
・ブラウザやNode.jsで実行可能になる
このとき、型チェックも同時に行われる

### 型とは何か？
型とは、データの種類を表す情報のこと

[例]
・数値（number）
・文字列（string）
・真偽値（boolean）

型の扱いの違い
動的型付け
・型指定が不要
・エラーは実行時に発覚

静的型付け
・事前に型を指定
・コンパイル時にエラーを検出
TypeScriptは静的型付けを採用しています。

TypeScriptの主な型
基本的な型
number
string
boolean
null / undefined など

よく使う型の種類
・リテラル型：特定の値だけを許可
・ユニオン型：複数の型のいずれか
・配列型：同じ型のデータをまとめる
・タプル型：要素数と型が固定された配列
・オブジェクト型：キーと値の構造を定義
・特殊型：any / unknown / void / never など

### TypeScriptの主な特徴
型エイリアスとインターフェイス
・type：型に名前をつけて再利用
・interface：オブジェクトの構造を定義（拡張可能）

クラス
・オブジェクト指向の考え方
・プロパティやメソッドをまとめて定義できる

型ガード
・typeof：実行時に型を判定
・keyof：オブジェクトのキーを型として扱う

ジェネリクス
・<T> を使った柔軟な型指定
・再利用性の高い関数・クラスを作れる

モジュール機能
・export / import を使ってファイル分割
・コード管理がしやすくなる

### TypeScriptを導入するメリット
開発効率の向上
・エディタの自動補完が強化される
・コーディングミスが減る

バグの早期発見
・コンパイル時にエラーを検出
・実行前に問題を防げる

可読性・保守性の向上
・型情報によりコードの意図が明確
・チーム開発での認識ズレを防げる
