### TypeScriptとは
・TypeScriptは、JavaScriptに「型」の仕組みを追加したプログラミング言語
・Microsoftが開発したオープンソースの言語で、JavaScriptのスーパーセット（上位互換）として設計されている

### 型を導入するメリット
・TypeScriptでは型を導入することで静的型チェックが可能になる
・これにより、エラーの防止やコードの安全性・可読性が向上する

### JavaScriptとの違い
・JavaScriptは動的型付け言語のため、実行時まで型エラーに気づきにくいという特徴がある
・一方、TypeScriptは静的型付け言語であり、コンパイル時に型エラーを検出できる
・その結果、予期せぬエラーを防ぎやすく、保守しやすいコードを書くことができる

### 利用範囲と互換性
・TypeScriptは、フロントエンド（React・Vueなど）やバックエンド（Node.js）など、幅広い開発で使われている
・TypeScriptで書かれたコードは最終的にJavaScriptにコンパイルされ、JavaScriptのすべての機能を利用できる
・JavaScriptと高い互換性があり、既存のJavaScriptからの移行やライブラリの利用もスムーズに行える







2. 基本概念
### TypeScriptの基本
・TypeScriptは、JavaScriptとは異なり、コードをそのまま実行することができない
・TypeScriptで書かれたコード（.tsファイル）は、コンパイルされてJavaScript（.jsファイル）に変換される必要がある

### コンパイルとは
・コンパイルとは、プログラムのコードを別の形式に変換するプロセスのことです。
・TypeScriptの場合、.tsファイルとして書かれたコードは、.jsファイルに変換されます。
・この変換によって、ブラウザやNode.jsなどのJavaScript実行環境で動作するようになります。

### 型チェックの仕組み
・TypeScriptのコンパイルでは、JavaScriptへの変換と同時に型チェックも行われます。
・つまり、開発中に型の間違いを発見し、エラーとして警告を出してくれます。

### コンパイルの例

・TypeScriptのコード例
```
let message: string = "Hello, TypeScript!";
console.log(message);
```

### このコードをコンパイルすると、以下のようなJavaScriptに変換される
```
var message = "Hello, TypeScript!";
console.log(message);
```

型情報である :string はJavaScriptには存在しないため、コンパイル時に削除されます。
その結果、TypeScriptのコードがJavaScriptとして実行できるようになります。

### そもそも型とは

プログラミングにおいて型とは、「データがどのような種類を持つかを表す情報」のことです。
例えば、「数値」や「文字列」など、データにはさまざまな種類があり、それらを分類するために型が存在しています。

### 動的型付けと静的型付け

プログラミング言語には、「動的型付け」と「静的型付け」の2つの型の扱い方がある

#### 動的型付け

変数にどんなデータでも代入でき、型の指定が不要

型の間違いが実行時まで発覚しない
```
let message = "Hello"; // 文字列
message = 123;        // エラーにならない
```

#### 静的型付け

変数や関数の型を事前に決める必要がある

コンパイル時に型チェックが行われる
```
let message: string = "Hello"; // 文字列と指定
message = 123;                // エラー: number型は代入できない
```
